# Grimoire Cycle Focuses

## Purpose

This note defines the first focused progression coordinate system for grimoires.

The goal is to keep skill progression **laser-scalpel focused**. Grimoires should not become a mushy skill buffet. Every grimoire progression path should be grounded in a small number of strong axes:

```text
Discipline + Cycle + Focus
```

This lets future skills, farming choices, character traits, and hybrid breakthroughs all point at the same progression grammar.

---

## Core Disciplines

Starting grimoires are organized around three primary disciplines:

| Discipline | Armor Affinity | Identity |
|---|---:|---|
| Brawn | Plate | Force, endurance, direct impact, battlefield authority |
| Cunning | Leather | Tempo, exploitation, status, snares, precision |
| Wisdom | Cloth | Observe, magic, support, cleanse, field control |

Armor affinity is presentation/progression metadata, not necessarily a hard combat restriction.

---

## Cycle Philosophy

Each cycle changes what a discipline means.

| Cycle | Theme | Progression Meaning |
|---|---|---|
| Cycle I: Origin | Foundation | Learn the obvious form |
| Cycle II: Resonance | Interaction | Learn rhythm, combination, and feedback |
| Cycle III: Conflict | Pressure | Learn exploitation, counters, and control under stress |
| Cycle IV: Singularity | Rule-breaking | Rewrite the battlefield around the discipline |

---

## Full Focus Grid

| Discipline | Cycle I: Origin | Cycle II: Resonance | Cycle III: Conflict | Cycle IV: Singularity |
|---|---|---|---|---|
| Brawn / Plate | Force | Momentum | Domination | Gravity |
| Cunning / Leather | Opportunism | Timing | Exploitation | Causality |
| Wisdom / Cloth | Perception | Attunement | Judgment | Authorship |

This grid is the backbone of grimoire progression.

Every future skill should be able to answer:

```text
What discipline?
What cycle?
What focus?
```

If a skill cannot answer those questions, it probably does not belong yet.

# Brawn / Plate

Brawn starts as force and endurance, then becomes battlefield authority.

## Cycle I: Origin — Force

The first lesson is that mass matters.

### Player Fantasy

```text
I hit the thing. I survive the thing.
```

### Mechanical Focus

- Direct physical damage
- Guard / brace
- Basic armor mitigation
- Breaking frost, shields, or fragile states through physical contact
- Reliable basic attacks

### Visual Identity

- Impact
- Slash
- Crush
- Shock
- Stagger
- Metal weight

### Candidate Unlocks

- Heavy Thump
- Brace
- Shield Bash
- Cleave

Brawn learns that force has rhythm.

- Heavy rhythm

- Physical cadence

- Overhand Chain
- Staggering Blow
- Iron Rhythm
- Counterbrace

# Farming Focus Integration

The five farming choices remain stable across the game:

1. Brawn discipline farm
2. Cunning discipline farm
3. Wisdom discipline farm
4. Wealth and materials
5. Gear

But the discipline farms display and reward according to the current cycle focus.

## Example: Cycle I

```text
Choose where to farm:

1. Force Grounds
   Brawn / Plate: direct impact, endurance, basic physical power.

2. Opportunism Grounds
   Cunning / Leather: speed, snares, poison, weak-point attacks.

3. Perception Grounds
   Wisdom / Cloth: Observe, magic basics, mend, cleanse.

4. Spoils Route
   Wealth and materials.

5. Armory Route
   Gear.
```

## Example: Cycle III

```text
Choose where to farm:

1. Domination Grounds
   Brawn / Plate: protection, suppression, armor break.

2. Exploitation Grounds
   Cunning / Leather: interrupts, punish windows, status detonation.

3. Judgment Grounds
   Wisdom / Cloth: dispels, exposes, precise counters.

4. Spoils Route
   Wealth and materials.

5. Armory Route
   Gear.
```

# Aligned and Cross-Discipline Farming

The player’s grimoire discipline and selected farm focus together determine progression direction.

## Aligned Farming

Aligned farming means the farm focus matches the character’s grimoire discipline.

```text
Brawn grimoire + Brawn farm = aligned
Cunning grimoire + Cunning farm = aligned
Wisdom grimoire + Wisdom farm = aligned
```

Aligned farming should provide the fastest core progression.

### Rule

```text
Matched farming gives efficiency.
```

## Cross-Discipline Farming

Cross-discipline farming means the character farms a discipline outside their grimoire.

```text
Brawn grimoire + Wisdom farm = cross-discipline
Wisdom grimoire + Cunning farm = cross-discipline
Cunning grimoire + Brawn farm = cross-discipline
```

Cross-discipline farming should be slower, but it should build toward hybrid breakthrough rewards.

### Rule

```text
Mismatched farming gives synthesis.
```

Player-facing language should avoid calling this “wrong.” Better terms:

- Cross-Discipline Study
- Hybrid Study
- Synthesis Route
- Breakthrough Route

# Hybrid Breakthrough Coordinates

A future hybrid breakthrough should key off:

```text
Home Discipline + Farm Discipline + Cycle
```

This means the same “wrong” farm can produce different rewards depending on current cycle.

## Cycle I Hybrid Examples

Simple bridges:

| Combination | Possible Breakthrough |
|---|---|
| Brawn + Wisdom | Runic Strike |
| Brawn + Cunning | Hooking Blow |
| Cunning + Wisdom | Alchemical Spark |

## Cycle II Hybrid Examples

Rhythm and interaction bridges:

| Combination | Possible Breakthrough | Concept |
|---|---|---|
| Brawn + Wisdom | Resonant Impact | Physical hit strengthens a ward or field |
| Brawn + Cunning | Hammerbeat Feint | Physical chain steals tempo |
| Cunning + Wisdom | Hex Clock | Snare / poison timing becomes magical detonation setup |

## Cycle III Hybrid Examples

Conflict and counterplay bridges:

| Combination | Possible Breakthrough | Concept |
|---|---|---|
| Brawn + Wisdom | Wardbreaker | Physical domination dispels or exposes |
| Brawn + Cunning | Predator Challenge | Taunt that punishes enemy action choices |
| Cunning + Wisdom | Surgical Hex | Exploit status to seal or interrupt enemy techniques |

## Cycle IV Hybrid Examples

Reality-bending bridges:

| Combination | Possible Breakthrough | Concept |
|---|---|---|
| Brawn + Wisdom | Law of Weight | Field rule around mass, mitigation, and forced impacts |
| Brawn + Cunning | Gravity Ambush | Delayed trap collapses enemy tempo into a crushing hit |
| Cunning + Wisdom | Causal Thesis | Conditional outcome rewrite |

# Character Trait Interaction

Character traits should bias farming and autotree decisions.

Traits should affect **path selection**, not just stats.

## Core Rule

```text
A trait should explain why two characters with the same grimoire make different farming choices.
```

## Example Traits

### Disciplined

Prefers the current cycle focus of its own discipline.

```text
Brawn / Plate in Cycle III -> favors Domination Grounds.
```

### Iconoclast

Prioritizes cross-discipline farming, especially where the current cycle has strong hybrid breakthrough potential.

```text
Cunning / Leather in Cycle III -> may choose Judgment Grounds for Cunning/Wisdom Conflict breakthroughs.
```

### Opportunist

Looks for Wealth or Gear unless a near-threshold discipline breakthrough is available.

```text
Wisdom / Cloth with weak gear -> may choose Armory Route before deeper Wisdom study.
```

### Adaptive

Switches to whatever farm answers recent losses.

```text
Brawn / Plate loses to poison repeatedly -> may choose Cunning Grounds or Gear Route.
```

Mara chose Domination Grounds because Disciplined favors aligned Cycle III Brawn growth.

# Suggested Data Model

## Discipline

```cpp
enum class Discipline {
  Brawn,
  Cunning,
  Wisdom,
};
```

## Cycle

```cpp
enum class Cycle {
  Origin,
  Resonance,
  Conflict,
  Singularity,
};
```

## Farm Focus

```cpp
enum class FarmFocus {
  Brawn,
  Cunning,
  Wisdom,
  WealthMaterials,
  Gear,
};
```

## Grimoire Focus

```cpp
struct GrimoireFocus {
  Discipline discipline;
  Cycle cycle;
  std::string name;
  std::string short_description;
};
```

## Lookup

```cpp
GrimoireFocus get_grimoire_focus(Cycle cycle, Discipline discipline);
```

## Farm Plan

```cpp
struct FarmPlan {
  Discipline grimoire_discipline;
  FarmFocus farm_focus;
  Cycle cycle;
};
```

## Farm Plan Kind

```cpp
enum class FarmPlanKind {
  AlignedDiscipline,
  CrossDiscipline,
  WealthMaterials,
  Gear,
};
```

## Cross-Discipline Key

```cpp
struct HybridBreakthroughKey {
  Discipline home_discipline;
  Discipline farm_discipline;
  Cycle cycle;
};
```

# Implementation Guidance

This should be implemented as metadata first.

Do not add a full skill tree yet.
Do not add a full economy yet.
Do not add gear generation yet.
Do not add a hybrid breakthrough tree yet.

The first pass should create the coordinate system and make it testable.

## First Implementation Goals

- Define all 12 grimoire focuses.
- Add lookup by `Cycle + Discipline`.
- Let farm display metadata resolve the current cycle focus.
- Classify farm plans as aligned, cross-discipline, wealth/materials, or gear.
- Represent future hybrid breakthrough coordinates as `Home Discipline + Farm Discipline + Cycle`.

---

# Acceptance Criteria

- Each discipline has exactly one focus per cycle.
- The 12-focus grid is test-covered:

```text
Origin:      Force / Opportunism / Perception
Resonance:   Momentum / Timing / Attunement
Conflict:    Domination / Exploitation / Judgment
Singularity: Gravity / Causality / Authorship
```

- Discipline farm display/progression metadata can resolve the current cycle focus.
- Cross-discipline farming can be classified by home discipline, farm discipline, and cycle for future breakthrough rewards.
- No unrelated progression or economy systems are introduced.
